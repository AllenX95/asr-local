# 紧凑任务中心设计

## 目标

重构一键工作流页面右侧的任务区域，减少队列与详情的重复信息，同时让用户看到可信、具体的任务进程。新增持久化的任务记录清除能力，但保留所有转录、总结和其他输出文件。

## 当前问题

- “任务队列”和“任务详情”分成两个面板，重复展示任务名、状态和刷新操作。
- 所有终态任务长期混在队列中，并与历史页面职责重叠。
- 列表突出 attempt 和 sequence 等内部字段，却没有展示排队位置、百分比、音频处理时长和更新时间。
- 工作流并发上限硬编码为 3，未使用 runtime capabilities。
- 控制和刷新操作缺少请求中反馈；窄窗口下工作流页面仍强制双栏。
- 系统没有删除工作流记录的持久化接口，仅在前端隐藏会在刷新或重启后恢复。

## 信息架构

右侧合并为一个“任务中心”面板。

1. 顶部显示运行中和排队中的数量，并保留唯一的刷新入口。
2. 活跃任务优先显示，包含 queued、running、paused 和 waiting_for_secret。
3. 选中任务在列表中原位展开，不再创建重复的详情面板。
4. 终态任务进入可折叠的“最近任务”区域，包含 completed、failed、cancelled 和 interrupted。
5. 产物、事件时间线和内部诊断字段放入选中任务内的二级折叠区。

## 任务列表

每个紧凑任务行显示：

- 显示名称和状态徽标。
- 整体进度条及百分比；没有可信进度时显示“等待进度数据”。
- 排队任务显示队列位置；运行任务显示当前阶段；终态任务显示完成、失败、取消或中断状态。
- 更新时间使用易读的相对表达或本地时间。

列表不再直接显示 attempt number、sequence 或原始事件类型。

## 展开的详细进程

选中任务原位展开以下信息：

- 整体进度 `overall_ratio`。
- 当前阶段进度 `stage_ratio`。
- 当前阶段的本地化名称。
- 队列位置 `queue_position`。
- 已处理音频时长和总音频时长；字段来自 worker 的真实进度数据。
- 当前步骤说明；优先使用 worker 事件携带的 detail。
- 快照更新时间。
- 暂停、继续、取消和重试操作。

前端不推算或伪造 worker 未提供的数据。字段缺失时使用明确的不可用文案。

## 清除语义

- 只有 completed、failed、cancelled 和 interrupted 状态允许清除。
- queued、running、paused 和 waiting_for_secret 必须先取消并进入终态。
- 清除前显示二次确认。
- 清除永久移除 workflow registry 中的任务记录及其事件记录。
- 清除不删除任何任务产物、音频源文件、转录、总结或用户编辑版本。
- 清除当前选中任务后，界面选择下一个可见任务；没有任务时回到空状态。

## 跨层接口

在 `WorkflowRuntime` 增加清除方法，并由 fake 与 Tauri adapter 实现。Tauri command 将请求传递给 Rust workflow v2 client，再由 Python supervisor/registry 执行持久化删除。接口必须校验任务已处于终态；非终态请求返回明确的领域错误。

刷新全部任务时，store 以 runtime 返回列表为权威来源，移除服务端已不存在的记录，避免已清除任务残留。

## 进度数据流

Python worker 在工作流进度事件中保留并上送以下规范字段：

- `overall_ratio`
- `stage_ratio`
- `queue_position`
- `processed_ms`
- `total_ms`
- `detail`

Rust 合约层透明传递这些字段；TypeScript 将 `progress` 从宽泛的 `Record<string, unknown>` 收紧为具名、可选字段。旧快照缺少新字段时仍可正常读取。

## 交互状态与响应式布局

- 刷新期间刷新按钮显示进行中状态并阻止重复点击。
- `pending_action` 非空时禁用冲突控制，并显示“暂停中”或“取消中”。
- 清除期间仅禁用对应任务的清除按钮。
- 低于现有桌面断点时，`workflow-grid` 切换为单列，任务中心移动到表单下方。
- 任务列表设置合理的最大高度和内部滚动，避免任务数量挤压整个页面。
- 并发上限与摘要来自 runtime capabilities，不再硬编码。

## 错误处理

- 控制、刷新和清除错误显示在对应任务附近；初始化或目录加载错误仍显示在页面级错误区。
- 后端拒绝清除活跃任务时，界面保持原记录并提示必须先取消。
- 清除成功但产物仍存在是预期行为，不显示为警告。

## 验证

- 为 store 增加刷新权威同步、清除选中项和清除失败行为测试。
- 为 fake runtime 和工作流 reducer/adapter 补充必要测试。
- 为 Python registry/supervisor 增加终态清除、活跃任务拒绝和产物保留测试。
- 运行 `npm run test`、`npm run typecheck` 和 `npm run build`。
- 运行相关 Python 测试与 `cargo test`/`cargo check`。
- 启动前端或桌面应用，在宽屏与窄窗口下截图检查任务中心布局、展开进度、控制 pending 状态和清除确认。

## 非目标

- 不删除或移动模型权重。
- 不删除任务产物和源音频。
- 不重新设计历史页面、工作流创建表单或 Python 调度算法。
- 不为缺失的进度字段估算虚假数据。
